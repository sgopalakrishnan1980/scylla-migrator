Known Issues & Workarounds

From scylla-migrator GitHub issues

Expand workarounds

Alternator: "INDEXES consumed capacity is not supported"

Set removeConsumedCapacity: true on the DynamoDB target (default in this UI). Required for Scylla Alternator 2024.2+.

#279 #249

Alternator PROVISIONED: "provisionedThroughput.readCapacityUnits is missing"

Create the target table manually with --provisioned-throughput ReadCapacityUnits=1,WriteCapacityUnits=1 before migrating.

#244

DynamoDB S3 Export: Savepoints not supported

Savepoints may show "Unexpected partition type" errors when source is dynamodb-s3-export. Migration still completes; resume is limited.

#247

Alternator with self-signed SSL

trustCA / CA certificate support for Alternator target is not yet implemented. Use HTTP or wait for #245.
